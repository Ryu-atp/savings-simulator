## 1. アプリケーション概要

「資産形成シミュレーター」は、日々の節約額を投資に回した場合に、将来どれほどの資産に成長するかを直感的にシミュレートできる React Native / Expo 製のモバイルアプリケーションです。

### 主な特徴と提供価値
*   **シンプルな入力と自動計算**: 積立額、年利、期間を入力するだけで、将来の資産額を即座に計算。
*   **自動保存と履歴管理**: 計算結果はローカルに自動保存され、過去のシミュレーションと簡単に比較可能。
*   **NISA・税金シミュレーション**: 課税口座（税率20%）とNISA口座（生涯投資枠）の違いによる最終資産の差を積層棒グラフでビジュアル化。
*   **目標からの逆算**: 目標達成に必要な月々の積立額を年齢ベースで逆算。
*   **マネタイズ機能**: サブスクリプション（RevenueCat）によるプレミアム機能提供と、Google AdMob によるバナー広告、証券口座等のアフィリエイトリンクの紹介。

## 4. シミュレーションおよび計算ロジック仕様

### 4.1 目標金額からの逆算ロジック (`calculator.js`)
目標金額（$FV$）、想定年利（$R\%$）、積立期間（$Y = \text{達成目標年齢} - \text{現在の年齢}$）から、必要な月々の積立額（$PMT$）を算出します。

*   **金利 $R = 0\%$ の場合**:
    $$PMT = \lceil \frac{FV}{Y \times 12} \rceil$$
*   **金利 $R > 0\%$ の場合**:
    月利 $r = \frac{R}{100 \times 12}$ 、積立月数 $n = Y \times 12$ とし、期末払い積立複利の公式から逆算します。
    $$PMT = \lceil \frac{FV \times r}{(1 + r)^n - 1} \rceil$$
    ※算出された値は円単位で切り上げ（`Math.ceil`）処理を行います。

---

### 4.2 資産額シミュレーション（税金区分対応） (`calculator.js`)
初期資産（$A_0$）、1回あたりの積立額（$P$）、想定年利（$r_{\text{annual}} = R / 100$）、運用年数（$Y$）、および積立頻度（$f$）に応じた将来価値を計算します。
積立頻度 $f$ に応じた年間積立ステップ数 $k$ は以下の通りです。
*   毎月 (`monthly`): $k = 12$
*   毎週 (`weekly`): $k = 52$
*   毎日 (`daily`): $k = 365$
*   毎年 (`yearly`): $k = 1$

#### A. 税金なし (`none`)
*   **毎年積立 (`yearly` / $k=1$) の場合**:
    $$A_t = A_0 \times (1 + r_{\text{annual}})^t + P \times \frac{(1 + r_{\text{annual}})^t - 1}{r_{\text{annual}}}$$
*   **それ以外の積立頻度（1期間あたりの金利 $r_{\text{period}} = r_{\text{annual}} / k$, 総期間数 $N = t \times k$）の場合**:
    $$A_t = A_0 \times (1 + r_{\text{annual}})^t + P \times \frac{(1 + r_{\text{period}})^N - 1}{r_{\text{period}}}$$
    ※ $r_{\text{period}} = 0$ の場合は単純に $P \times N$ を加算。

#### B. 税金（20％） (`tax`)
毎年発生する「運用益」に対し、年次で20%を税金として差し引く複利計算です。
各年 $t$ において、
1.  年初の資産額を $V_{t-1}$ とする。
2.  その年の積立による期末時点での税引前資産額 $V'_{t}$ を計算：
    *   毎年積立: $V'_t = V_{t-1} \times (1 + r_{\text{annual}}) + P$
    *   その他: 年間ステップ数 $k$ 回分、各ステップで $V \leftarrow V \times (1 + r_{\text{period}}) + P$ を繰り返す。
3.  年間の税引前増加額（運用益）を計算：
    $$\text{Gain}_t = V'_t - V_{t-1} - (P \times k)$$
4.  運用益がプラスの場合、税金を差し引く：
    $$V_t = \begin{cases} V'_t - (\text{Gain}_t \times 0.20) & (\text{Gain}_t > 0) \\ V'_t & (\text{Gain}_t \le 0) \end{cases}$$
5.  初期資産単体の複利成長（`assetGrowth`）も同様に年次で20%の課税差し引きを行いながら追跡します。

#### C. NISA利用 (`nisa`)
累計投資元本が **1,200万円** を超えた分のみ課税口座（税率20%）で運用されるシミュレーションです（※実装上の注意点あり、詳細は後述）。
*   資産を「NISA口座残高（非課税）」と「特定口座残高（課税）」に分離して管理。
*   累計投資元本（初期資産 $A_0$ ＋ これまでの積立額の累計）を計算し、1,200万円に達するまではすべてNISA口座に新規資金を投入。
*   1,200万円の上限を超えた新規積立分は、特定口座に投入。
*   年次計算において：
    *   NISA口座の運用益：非課税（税引処理なし）。
    *   特定口座の運用益：年間運用益に対して20%の税金を引き去る。

## 5. 技術仕様とデータ構造

### 5.1 状態管理 (`AppContext.js`)
アプリ全体のグローバルステートは React Context API で管理されています。

*   **主要な State**:
    *   `isPremium` (boolean): プレミアムプランの購読状態。
    *   `history` (Array): 計算履歴データの配列。
    *   `affiliateUrl` (string | null): 生存確認された有効なアフィリエイトURL。
    *   `isLoading` (boolean): AsyncStorage等からのローカルデータロード中フラグ。
    *   `currentAsset` (number): ユーザーの現在の資産額（初期資産として計算に使用）。
    *   `userAge` (number): ユーザーの年齢。

---

### 5.2 データ永続化 (AsyncStorage) (`storage.js`)
全データはローカルストレージ（AsyncStorage）に以下のキーで保存され、外部サーバー通信は行いません（サーバー維持コスト・課金リスクがゼロの設計）。

| キー | 保存形式 | 用途 |
| :--- | :--- | :--- |
| `@savings_history` | JSON (Array) | 計算履歴のリスト（お気に入りフラグ含む） |
| `@subscription_info` | JSON (Object) | サブスクリプション状態キャッシュ（`{ isPremium, expiresAt }`） |
| `@current_asset` | String (Number) | ユーザーが設定した「現在の資産額」 |
| `@user_age` | String (Number) | ユーザーが設定した「現在の年齢」 |

#### 計算履歴スキーマ（`HistoryEntry`）
```typescript
interface HistoryEntry {
  id: string;               // 一意のID (タイムスタンプ + ランダム文字列)
  savedAt: string;          // 保存日時のISO文字列

  inputs: {
    principal: number;      // 1回あたりの積立額（節約額）
```

### 5.4 アフィリエイトURL生存確認 (`linkChecker.js`)
アフィリエイト報酬の損失を防ぐため、起動時にアフィリエイトURLの生存確認をバックグラウンドで行います。

*   **動作仕様**:
    *   `Constants.expoConfig?.extra?.affiliate` から `primaryUrl` (推奨リンク) と `fallbackUrl` (楽天証券等の予備リンク) を取得。
    *   `primaryUrl` に対し、最軽量の `HEAD` リクエストを送信して死活監視を実施。
    *   タイムアウト値は **5秒**。
    *   レスポンスが 200-299、または 405 (HEADリクエスト未許可だがURLは生存) の場合は `primaryUrl` を「有効」と判定。
    *   エラー、タイムアウト、4xx/5xx レスポンスの場合は自動的に `fallbackUrl` にフォールバック。
    *   結果はメモリ上にキャッシュされ、起動中に重複してリクエストが飛ぶのを防止しています。

## 6. 【重要】ソースコード解析による指摘・改善推奨事項

ソースコードを詳細に解析した結果、**ドキュメント上の仕様と実際のコードの不一致**、および**不具合（バグ）の疑いがある箇所**が発見されました。プロジェクトマネージャーへ報告し、修正を検討することを強く推奨します。

### 指摘 1: NISA生涯投資枠の上限値の不一致（解決済み）
*   **以前の仕様書（docs/specification.md）の記述**:
    *   「NISA口座（**生涯投資枠1,200万円**（旧記述：1,800万円））の違いによる最終資産の差をビジュアル化」
